# 场景
描述问题背景、目标用户和最终结果。

# 目标
- 清晰列出完成后应该具备的能力或产出。
- 说明涉及的系统或约束。

# 前提条件
- 列出先决条件、账号、依赖工具。
- 提供快速验证步骤。

# 步骤
1. 步骤标题 — 简短说明。
   - 命令或操作。
   - 验证点。
2. 重复直到完成。

# 验证
- 如何确认执行成功。
- 自动化或脚本命令。

# 扩展
- 可选优化或相关链接。
- 后续深入方向。
